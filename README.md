Project Folder Structure

This project uses a simple and organized folder structure to make the system easier to understand and manage.

Folders:

src/
This folder contains the main source code of the system.

docs/
This folder contains documentation such as project descriptions, notes, and instructions.

assets/
This folder contains images, files, or other resources used in the project.

README.md
This file explains the purpose of the project, the folder structure, and the architecture used.

Purpose of the Folder Structure

The purpose of this folder structure is to keep the project organized.
Each folder has its own function so that the files will not be mixed together.

This makes the project:

easier to maintain
easier to understand
easier to develop as a group
Component Layering Strategy

The system follows a simple 3-layer architecture:

Presentation Layer
This layer is responsible for the user interface (UI).
It is where the user interacts with the system.
Example:

input forms
buttons
display of information
Business Logic Layer
This layer contains the main functions and rules of the system.
It processes the data given by the user and performs the required operations.
Example:

processing customer information
calculating results
validating inputs
Data Layer
This layer is responsible for storing and retrieving data.
It manages how data is saved and accessed in the system.
Example:

saving files
reading stored data
updating records
